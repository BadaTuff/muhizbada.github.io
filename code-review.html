CS 499 Milestone One: Informal Code Review

Code Review Video

In this code review, I walk through the existing code from two artifacts: the WeightAppUI Android application and the Travlr Getaways full-stack web application. I analyze each artifact's structure and functionality, identify weaknesses and limitations, and explain the specific improvements I planned to make across three enhancement categories: Software Design and Engineering, Algorithms and Data Structures, and Databases.

Code Review Summary

Artifacts Reviewed

WeightAppUI (CS 360) - An Android mobile application for tracking daily weight measurements. Used for both the Software Design/Engineering and Algorithms/Data Structures enhancement categories.

Travlr Getaways (CS 465) - A full-stack travel booking web application built with the MEAN stack. Used for the Databases enhancement category.

Key Findings

Software Design and Engineering: Identified plain text password storage, insufficient input validation, tight coupling between UI and business logic, and manual cursor handling for database queries.

Algorithms and Data Structures: Identified lack of sorting functionality, inefficient searching with no filtering capability, suboptimal data structure choices for lookup operations, and numeric data stored as strings.

Databases: Identified lack of pagination in API responses, absence of strategic database indexing, limited query functionality with no search or filtering, and no analytics or aggregation capabilities.
